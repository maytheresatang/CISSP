⚠️ Exam Trap: Who sets classification? The data owner sets the classification — not the custodian, not IT, not the security team. Classification is a business decision.
Classification Criteria
  • Value — What is the asset worth to the organisation?
  • Age — Does sensitivity decrease over time?
  • Useful life — When does it become irrelevant?
  • Personal association — Does it identify an individual?

Three States of Data

💾 Data at Rest

Stored on media (disk, tape, USB, cloud storage, database).

Controls: Full disk encryption (FDE), database encryption, access control, physical security, media sanitisation.

Examples: AES-256 FDE, TDE (Transparent Data Encryption), encrypted backups

📡 Data in Transit

Moving across a network (LAN, WAN, internet, VPN).

Controls: TLS/SSL, IPsec VPN, HTTPS, S/MIME (email), SSH.

Attack surface: eavesdropping, MITM, packet capture

🖥️ Data in Use

Being processed in RAM/CPU. Hardest to protect — data must be decrypted to process.

Controls: Trusted Execution Environment (TEE), Intel SGX, memory encryption, access controls, DLP, process isolation.

Emerging: Confidential Computing addresses this gap

⚠️ Exam Trap: Which control matches which state? TLS protects data in transit. AES-256 on disk protects data at rest. Neither protects data in use. The exam tests whether you know which control matches which state.
Data Lifecycle
  1. Create — Data is generated or collected. Classification should be assigned here.
  2. Store — Saved to media. Encryption, access control, backup apply.
  3. Use — Accessed and processed. Least privilege, DLP, audit logging apply.
  4. Share — Transmitted internally or externally. TLS, DRM, NDA, classification labels apply.
  5. Archive — Long-term storage. Retention policy, media degradation checks.
  6. Destroy — End of useful life. Sanitisation methods apply (see Destruction tab).
Retention Policies

Retention policy defines how long data must be kept before it can be destroyed. Driven by:

  • Legal/regulatory requirements (GDPR Article 5: kept no longer than necessary)
  • Business need (contracts, financial records)
  • Litigation hold (legal hold supersedes normal retention — do not destroy)
Baseline, Scoping, and Tailoring
ConceptDefinition
BaselineMinimum set of security controls for a given environment type. Starting point before tailoring.
ScopingLimiting which controls apply to a specific system (removing irrelevant ones). e.g., wireless security baseline doesn't apply to a wired-only server.
TailoringAdjusting baseline controls to fit the specific environment (tighten or loosen). Must be documented and justified.
SupplementationAdding controls beyond the baseline for higher-risk environments.
Data Remanence

Data remanence = residual data remaining on media after deletion or formatting. Deleted ≠ destroyed. The OS marks space as available but data remains until overwritten.

Sanitisation Methods by Media Type
MethodDescriptionWorks On
Clearing (Overwriting)Write patterns of 0s and 1s over data. Media reusable. DoD 5220.22-M standard (7 passes).HDDs, magnetic media
Purging / DegaussingExpose to strong magnetic field — destroys magnetic domains. Media unusable after.Magnetic HDDs, tapes — NOT SSDs/flash
Destruction / PhysicalShred, incinerate, disintegrate, pulverise. Most thorough — media unrecoverable.All media types
Cryptographic ErasureDestroy the encryption key — data unreadable even if media remains. Fast.Encrypted SSDs, cloud storage
🚨 Critical Exam Trap: Degaussing SSDs Degaussing does NOT work on SSDs or flash memory. These use electrical charge, not magnetic fields. The correct method for SSDs is cryptographic erasure or physical destruction.
⚠️ Hierarchy for "most secure" destruction When the exam asks for the most secure method: Physical Destruction > Degaussing (magnetic) / Crypto Erasure (SSD) > Purging > Clearing. Physical destruction is always the gold standard answer.
Object Reuse

Before reallocating storage media (RAM, disk, tapes) to another user or process, ensure all previous data is properly sanitised. This is object reuse — a key control in multi-user environments.
