Digital Forensics — Order of Volatility (CRITICAL)
🚨 Highest Risk Gap in Domain 7 Collect most volatile evidence FIRST. Once power is lost, volatile evidence is gone forever.
Order of Volatility — Most → Least Volatile
Registers → RAM → Swap/Page → Disk → Remote → Archive
1. CPU registers, cache
2. RAM (running processes, network connections, crypto keys)
3. Swap / Page file (virtual memory on disk)
4. Persistent disk storage (files, logs, databases)
5. Remote logging / monitoring data
6. Archival media (tapes, backups)

Logging and Monitoring

⚠️ NTP is a forensic prerequisite If NTP fails or is compromised, log timestamps become unreliable and forensic timelines break. NTP (port 123) is not just an operational nicety — it is a security control.

Need-to-Know and Least Privilege

Need-to-know: Access to information only when required to perform assigned duties. Even users with high clearance do not automatically have access to all data at that level — they must have a need.

Least privilege: Grant only the minimum access rights required. Applies to users AND systems (service accounts, processes).

Separation of Duties in Operations

Critical operational tasks require multiple individuals to complete. Prevents any single person from controlling a full process — reduces fraud and error risk.

  • Operator who runs backup ≠ operator who validates the backup
  • Developer who writes code ≠ person who promotes it to production
  • Person who approves a transaction ≠ person who executes it

Grid Computing (Architecture Distinction)
⚠️ Quiz Gap: Grid vs Cloud vs Cluster Grid computing = pooling idle CPU resources from geographically dispersed, independent machines for one large computational task. Not cloud (provisioned resources), not clustering (co-located high-availability). CISSP risk: grid controller compromise = security boundary breach across all contributing nodes.
Incident Response Phases (NIST SP 800-61)
  1. Preparation — Build the IR capability before incidents occur: team, tools, playbooks, contacts, communication plan
  2. Detection & Analysis — Identify potential incidents via SIEM, alerts, user reports. Triage severity. Declare incident if threshold met.
  3. Containment — Short-term (isolate affected system now) + Long-term (stable, allow business to continue). Preserve evidence.
  4. Eradication — Remove malware, close vulnerabilities, clean affected systems
  5. Recovery — Restore systems from clean backups. Monitor closely. Validate integrity before returning to production.
  6. Post-Incident Activity (Lessons Learned) — Document what happened, what worked, what failed. Update playbooks.
⚠️ Exam Trap: Containment vs Eradication order Containment comes BEFORE eradication. You must stop the bleeding before you can clean the wound. Also: during containment, preserve forensic evidence before cleaning. Do not wipe systems before imaging.
ISC2 Ethics in Incident Response

If investigation reveals criminal activity, the security professional must prioritise public safety (Canon 1) over client confidentiality (Canon 3). Evidence must be preserved. Contact legal counsel before law enforcement to understand obligations.

Automatic Failover
⚡ Quiz Gap — Failover Terminology
You selected: "fail-safe" or "resilience" (broader terms)
✅ Correct: "Automatic failover" — the specific term for systems that automatically switch to a redundant system when the primary fails, without human intervention
TermDefinition
FailoverSystem automatically switches to backup when primary fails. No human action required.
FailbackReturn to primary after it is restored. May require manual steps.
Fail-safeOn failure, system defaults to a safe state (e.g., door unlocks on power loss). Life safety concept.
Fail-secureOn failure, system defaults to a secure/deny state (e.g., firewall blocks all traffic if it crashes).
ResilienceBroad ability to absorb disruption and recover. Failover is a specific resilience mechanism.

⚡ Quiz Gap Remediation — 5 Missed Questions

Quiz score: 10/15 (67%). Cross-cutting error pattern: choosing broad terms when the exam requires the most specific one, and not selecting the most complete risk-framing answer.

Gap 1 — Configuration vs Change Management
Wrong: "Change management" tracks and audits data centre additions
✅ Configuration Management — maintains the CMDB and tracks what is IN the environment. Change Management controls HOW changes are approved and implemented.
Gap 2 — Full scope of what patches do
Wrong: Patches only fix security vulnerabilities
✅ Patches fix security vulnerabilities, AND fix software bugs, AND can add or improve functionality. Always choose the most complete description.
Gap 3 — Automatic failover terminology
Wrong: "Fail-safe" or broad resilience term selected
✅ "Automatic failover" = the specific technical term for system switching to backup without human intervention. Fail-safe = safe state on failure (life safety). These are different concepts.
Gap 4 — TPM and Secure Boot protect against what malware type?
Wrong: "Rootkit" selected
✅ Bootkit — infects the Master Boot Record or UEFI, loading before the OS. TPM and Secure Boot verify the boot chain integrity specifically to block bootkits. Rootkits operate within the OS (after boot).
Gap 5 — Which RAID level always reduces capacity by 50%?
Wrong: RAID 5 or RAID 6 selected
✅ RAID 1 (Mirroring) — every bit of data is duplicated. Always 50% capacity regardless of number of drives. RAID 5 only loses the equivalent of ONE drive worth of capacity regardless of array size.
