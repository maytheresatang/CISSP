🚨 Nessus vs Nikto — Most Tested Tool Confusion Nessus = general network/host vulnerability scanner. Nikto = specifically web server testing (HTTP/HTTPS). If the scenario involves a web server on port 80 or 443 → Nikto. If it's a general network vulnerability scan → Nessus.

Penetration Test Phases
  1. Planning / Scoping — Define scope, get written authorisation (rules of engagement). Critical: no test without written permission.
  2. Reconnaissance (Recon) — Passive: OSINT, WHOIS, LinkedIn. Active: port scanning, banner grabbing.
  3. Scanning / Enumeration — Identify live hosts, open ports, services, OS. Tools: Nmap, Nessus.
  4. Exploitation — Attempt to exploit discovered vulnerabilities. Tools: Metasploit, custom exploits.
  5. Post-Exploitation / Lateral Movement — Escalate privileges, pivot to other systems, establish persistence.
  6. Reporting — Document findings, evidence, impact, and remediation recommendations.
  7. Cleanup / Remediation Verification — Remove artefacts. Re-test after fixes.
⚠️ Rules of Engagement (RoE) Written authorisation is NON-NEGOTIABLE before testing. Without it, penetration testing is criminal. RoE defines: scope (IP ranges, systems), timing (business hours vs off-hours), escalation contacts, off-limits systems.

SOC Reports (SSAE 18 / AICPA)
ReportAudienceWhat it coversType 1 vs Type 2
SOC 1Financial auditors of user organisationsControls over financial reporting (ICFR)Type 1 = point in time. Type 2 = period (6–12 months).
SOC 2Security-focused: customers, business partnersTrust Service Criteria: Security, Availability, Processing Integrity, Confidentiality, PrivacyType 1 = design. Type 2 = design + operating effectiveness over time.

📌 Practical Usage: SOC 2 Type 2 In your vendor assurance role, you request SOC 2 Type 2 reports — they show controls were operating effectively over a period (not just designed correctly). A Type 1 is weaker evidence for ongoing assurance.
⚠️ Exam Trap: SOC 1 vs SOC 2 SOC 1 = financial reporting controls. SOC 2 = security and operational controls. Never mix these up. A cloud hosting provider showing their security posture → SOC 2. A payroll processor showing financial control reliability → SOC 1.

AcronymFull NamePurpose

📌 CVE vs CVSS distinction CVE = the unique ID (name). CVSS = the score (severity number). CVE-2021-44228 is Log4Shell. Its CVSS score was 10.0 (critical). Know the difference.

Regression Testing

Regression testing = re-running a test suite after changes to verify previously passing tests still pass. Confirms new code changes don't break existing functionality. Essential in CI/CD pipelines.

KPI vs KRI — Critical Distinction
KPI (Key Performance Indicator)KRI (Key Risk Indicator)
MeasuresHow well controls and processes are performing nowLeading signal of increasing risk — predictive
Time orientationLagging indicator (historical)Leading indicator (forward-looking)
Example% of critical patches applied within 30 days = 94%Increase in failed login attempts = rising brute-force risk
Used byOperations team to show control effectivenessRisk function to identify emerging risk
Action when threshold breachedInvestigate control failureEscalate risk to risk committee, consider control enhancement
✅ Key Distinction In your role, you design KRIs for the risk dashboard and monitor KPIs reported by 1LoD. Map this directly to CISSP questions about security metrics design and reporting to senior management.
Synthetic Transactions

Synthetic transactions = scripted test transactions run against production systems to continuously verify availability and performance without real user traffic. Simulates what a user would do (login → search → checkout) and validates the full path is working.
